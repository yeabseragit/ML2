1. Data Preparation

  • The first step is to Load the data set from a CSV file (nhis_2022.csv). Remove rows where ASTATFLG isnt 1 (indicating valid records).and Replace any invalid values in MARSTCUR (marital status) and other variables with NA. -Convert the DIABETICEV (diabetes variable) into a factor with two levels (NoDisease and Disease).
    • Remove rows with any missing values using drop_na().
    • Finally, keep only relevant columns (DIABETICEV, AGE, BMICALC, HRSLEEP, MOD10DMIN, CIGDAYMO). The selected variables(predictors) AGE, BMICALC, HRSLEEP, MOD10DMIN, and CIGDAYMO were chosen based on both medical knowledge and statistical correlation with diabetes. AGE is a known risk factor, as the likelihood of developing diabetes increases with age. BMICALC (Body Mass Index) reflects body weight status; higher BMI is strongly associated with Type 2 diabetes. HRSLEEP represents sleep duration, which affects insulin sensitivity and metabolic health. MOD10DMIN measures physical activity, where lower exercise levels are linked to higher diabetes risk. CIGDAYMO, indicating cigarette use, is included because smoking contributes to insulin resistance and cardiovascular complications common in diabetics. Together, these variables capture key lifestyle and health factors known to influence diabetes risk.

# Load and clean data
nhis <- read_csv("C:/Users/Yeabs/Downloads/nhis_2022.csv") %>%
  # Keep only adults with valid response flags
  filter(ASTATFLG == 1) %>%
  
  # Recode values
  mutate(
    # Set invalid marital status codes to NA
    MARSTCUR = ifelse(MARSTCUR %in% 996:999, NA, MARSTCUR),
    
    # Recode diabetes variable
    DIABETICEV = factor(
      case_when(
        DIABETICEV == 2 ~ 1L,  # "Yes"  Disease
        DIABETICEV == 1 ~ 0L,  # "No"  NoDisease
        TRUE ~ NA_integer_
      ),
      levels = c(0, 1),
      labels = c("NoDisease", "Disease")
    ),
    
    # Set 996-999 as NA for predictors
    across(c(AGE, BMICALC, HRSLEEP, MOD10DMIN, CIGDAYMO), 
           ~ ifelse(.x %in% 996:999, NA, as.numeric(.x)))
  ) %>%
  
  # Keep only married individuals for consistency
  filter(MARSTCUR == 1) %>%
  
  # Drop any remaining rows with NA
  drop_na() %>%
  
  # Select final modeling variables
  select(DIABETICEV, AGE, BMICALC, HRSLEEP, MOD10DMIN, CIGDAYMO)

2. Train-Test Split & Scaling

  • We split the data into a training set (70%) and a testing set (30%). createDataPartition ensures that the data is representative (stratified sampling) for the target variable (DIABETICEV). We apply feature scaling using center (mean centering) and scale (standardization) to the training and test data to make sure the variables are on the same scale.
set.seed(123)
train_index <- createDataPartition(nhis$DIABETICEV, p = 0.7, list = FALSE)

test_data <- nhis[-train_index, ]

3. Model Training

Linear SVM with tuning

  • Here, we calculate the class imbalance in the training data. The weight for the Disease class is adjusted based on its frequency in the dataset to make the model account for class imbalance.

To address the issue of class imbalance in the outcome variable DIABETICEV, class weighting was applied during model training. Specifically, the training set exhibited a much higher number of NoDisease cases compared to Disease cases. To compensate for this imbalance, weights were assigned inversely proportional to class frequency.

The weight for the minority class (Disease) was calculated as the ratio of NoDisease to Disease counts in the training data. This approach ensures that misclassifying a rare case (Disease) incurs a higher penalty, helping the SVM model pay more attention to the minority class. These weights were supplied to the svm() function using the class.weights parameter during model training.

# Calculate class weights for imbalance
class_weights <- c(
  "Disease" = sum(train_scaled$DIABETICEV == "NoDisease")/nrow(train_scaled),
  "NoDisease" = sum(train_scaled$DIABETICEV == "Disease")/nrow(train_scaled)
)

# Linear SVM with tuning
set.seed(123)
tune_linear <- tune(

  data = train_scaled,
  kernel = "linear",
  ranges = list(cost = c(0.01, 0.1, 1, 10, 100)),
  class.weights = class_weights,
  probability = TRUE
)

4. Model Evaluation

eval_model <- function(model, test_data) {
  pred <- predict(model, test_data)
  prob <- attr(predict(model, test_data, probability = TRUE), "probabilities")[, "Disease"]
  
  roc_obj <- roc(test_data$DIABETICEV, prob, quiet = TRUE)
  
  list(
    confusion = confusionMatrix(pred, test_data$DIABETICEV, positive = "Disease"),
    roc = roc_obj,
    auc = auc(roc_obj)
  )
}

results <- list(
  Linear = eval_model(best_linear, test_scaled),
  Radial = eval_model(best_radial, test_scaled),
  Polynomial = eval_model(best_poly, test_scaled)
)
